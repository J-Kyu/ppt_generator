# UI/UX Workflow

사용자가 제시한 변경된 4단계 작업 Flow를 기반으로 한 UI/UX 명세입니다.

## 1단계: 진입 및 샘플 PPT 분석 (Onboarding & Upload)
- **화면 구성:** 화면 중앙에 커다란 '드래그 앤 드롭' 영역과 '파일 찾기' 버튼을 배치합니다.
- **사용자 경험:** 베이스가 될 샘플 PPT 파일(기본 템플릿)을 삽입합니다.
- **피드백 및 백그라운드 처리:** 로딩 스피너와 함께 "템플릿 레이아웃 분석 중..." 메시지를 노출합니다. 엔진이 원본 PPT의 **마스터 슬라이드와 슬라이드 레이아웃**을 순회하며 내부 구조와 텍스트 박스 등을 분석합니다.

## 2단계: 분석된 레이아웃 제공 (Layout Overview)
- **화면 구성:** 분석이 끝난 PPT 내의 모든 가용 '레이아웃' 목록을 사용자에게 제공합니다. 이때 각 레이아웃이 어떤 내용으로 구성되어 있는지(예: "타이틀형", "본문과 이미지형", 텍스트 상자 개수 등)를 간략한 내용 및 썸네일과 함께 보여줍니다.
- **사용자 경험:** 사용자는 자신이 지금 구성하고 싶은 문서의 타입(레이아웃)이 무엇인지 시각적으로 한눈에 파악할 수 있어야 합니다.

## 3단계: 슬라이드 구성 및 문구 추가 (Slide Construction)
- **화면 구성:** 창을 둘로 나누어, 한쪽에는 사용자가 2단계에서 **원하는 레이아웃을 클릭(선택)**할 수 있는 목록을 배치합니다. 다른 쪽에는 선택한 레이아웃 형태에 맞춰 변하는 **동적 입력 폼(텍스트 입력 칸)**을 보여주고, '슬라이드 추가' 버튼을 배치합니다.
- **사용자 경험:** 사용자는 원하는 레이아웃을 선택하고, 자신이 원하는 문구를 추가한 뒤 저장하여 하나의 슬라이드를 완성합니다. 이 과정을 여러 번 반복하여 전체 프레젠테이션 스토리를 "구성(Construct)"합니다. 구성된 슬라이드 목록은 화면 하단이나 측면에 리스트 형태로 누적되어 표시됩니다.

## 4단계: 최종 생성 및 내보내기 (Generate & Export)
- **화면 구성:** 전체 슬라이드 구성이 끝나면 우측 하단의 명확한 'PPT 생성하기 (Generate)' 버튼을 누릅니다.
- **사용자 경험:** 사용자가 레이아웃 기반으로 조합/입력한 문구를 토대로 완전히 새로운 PPT가 자동 합성됩니다. 완료 시 '성공' 애니메이션과 함께, 만들어진 파일을 바로 열거나 저장된 폴더를 탐색기로 열어주는 친절한 버튼이 제공됩니다.
