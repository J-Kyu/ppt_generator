# UI/UX Workflow

## 1단계: 진입 및 파일 업로드 (Onboarding & Upload)

화면 구성: 화면 중앙에 커다란 '드래그 앤 드롭' 영역과 '파일 찾기' 버튼을 배치합니다.

사용자 경험: 복잡한 메뉴 없이, 앱을 켜자마자 무엇을 해야 하는지(PPT 파일을 넣으라는 것) 직관적으로 알 수 있어야 합니다.

피드백: 파일이 인식되면 잠시 '로딩 스피너'와 함께 "PPT 레이아웃을 분석 중입니다..."라는 메시지를 띄워 진행 상황을 알립니다.

## 2단계: 분석 결과 확인 및 구조화 (Analysis Review)

화면 구성: 불러온 PPT의 슬라이드 목록이나 대표 레이아웃을 보여줍니다.

사용자 경험: 시스템이 인식한 텍스트 박스들(예: Title_1, Subtitle_1, Content_Body)이 올바른지 사용자가 가볍게 확인할 수 있는 단계입니다.

백그라운드 처리: 이 단계에서 내부적으로 JSON 스키마가 확정됩니다.

## 3단계: 동적 데이터 입력 (Data Entry - 핵심 화면)

화면 구성: 왼쪽 또는 상단에는 데이터 입력 폼(JSON 기반으로 자동 생성된 텍스트 박스들)이 있고, 오른쪽 또는 하단에는 변경 사항이 반영될 미리보기(Preview) 썸네일을 배치하는 것이 좋습니다.

사용자 경험: 사용자는 JSON이나 복잡한 구조를 전혀 알 필요 없이, 평범한 웹 회원가입 폼을 채우듯 빈칸에 텍스트만 입력하면 됩니다.

유효성 검사(Validation): 필수 항목(예: 메인 타이틀)을 비워두고 '생성'을 누르면, 해당 폼 영역에 붉은색 경고 메시지를 띄워 입력을 유도합니다.

## 4단계: 최종 생성 및 내보내기 (Generate & Export)

화면 구성: 'PPT 생성하기'라는 명확한 CTA(Call to Action) 버튼을 누르면 완료 화면으로 넘어갑니다.

사용자 경험: 성공 메시지와 함께 '완료된 파일 열기', '저장 폴더 열기', '새 작업 시작하기' 버튼을 제공하여 앱 바깥의 워크플로우(탐색기 열기 등)까지 배려합니다.
