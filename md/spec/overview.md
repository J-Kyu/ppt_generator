# Purpose
- 제공된 샘플 PPT 파일의 마스터 및 레이아웃 슬라이드를 분석하고, 사용자가 원하는 레이아웃을 직접 선택해 본인이 원하는 문구를 추가하여, 이를 바탕으로 완전히 새로운 구성의 프레젠테이션(PPT) 파일을 생성하는 서비스 앱입니다.

# Techinical Overview
## 추천 시스템 아키텍처: 파이프라인 기반 계층 구조
- 파일 처리 계층 (I/O Layer): 로컬 파일 시스템에서 PPTX 파일을 읽고 저장합니다. (python-pptx 활용)
- 분석 엔진 (Analysis Engine): 마스터 슬라이드 및 레이아웃을 분석하여 사용할 수 있는 템플릿 포맷과 입력 요소들을 추출합니다.
- 데이터 스토리지 (Local State): 사용자가 선택한 레이아웃 배열과 그 안에 작성한 텍스트 상태(새로운 슬라이드 덱)를 저장합니다.
- 동적 UI 레이어 (Interaction Layer): 선택한 레이아웃 구조를 바탕으로 사용자 입력 폼을 그리고, 사용자가 슬라이드를 조립(추가)할 수 있게 합니다.
- 렌더링 엔진 (Generation Engine): 사용자가 직접 구성한 레이아웃 순서와 입력 데이터들을 바탕으로 원본 템플릿에서 새 슬라이드들을 추가(Add Slide)해가며 최종 PPT를 빌드합니다.

## 단계별 핵심 기술 및 설계 포인트
### PPT 레이아웃 분석 (Analysis Engine)
- 일반 슬라이드가 아닌, `Slide Master`와 `Slide Layout`의 모양(names, placeholders 등)을 순회하며 고유 식별자와 속성을 딕셔너리로 맵핑합니다.
- 서비스 상에서는 텍스트 속성 조작을 기본으로 우선 지원합니다.

### 동적 폼 생성 도구 및 슬라이드 빌더 (Form UI)
- Pydantic/JSON Scheme 기반의 동적 폼을 화면에 뿌려줍니다.
- 사용자는 원하는 레이아웃을 리스트에서 선택하여 텍스트를 기입하고, 슬라이드 리스트(List) 배열에 순서대로 Push하게 됩니다.

### 데이터 직렬화 및 검증
- 생성 버튼 전 필수 입력 항목 누락 검증을 포함해야 합니다.

# Recommed Technical Stack
- GUI 프레임워크: Flet (빠른 개발)
- PPT 조작: python-pptx (거의 유일하고 강력한 표준)
- 데이터 관리: Pydantic (상태 관리 및 데이터 무결성 검증용)
- 패키징: PyInstaller
- Python: uv를 통한 python >= 3.13 버전 사용

# Dev Environment
- Mac과 Windows에 모두 개발 가능하다.

# UI/UX Workflow
전체 서비스 관점의 흐름은 다음과 같습니다.
1. 샘플 PPT 분석: 파일을 업로드하고 마스터/레이아웃을 분석한다.
2. 레이아웃 제공: 분석된 구조를 사용자가 알아보기 쉽게 간략한 요약과 함께 화면에 띄운다.
3. 슬라이드 구성: 사용자가 원하는 레이아웃 서식을 택하고 문구를 채워 넣어 개별 슬라이드의 연속(덱)을 구성한다.
4. 재생성: 구성된 슬라이드 배열을 기반으로 새 PPT를 렌더링한다.

# Precautious
- 폰트 및 리소스: 사용자가 지정하지 않은 시스템 외 폰트가 깨질 우려가 있으므로, 텍스트 삽입 시 원본 템플릿의 문단 서식(text_frame paragraph)을 최대한 훼손하지 않고 텍스트 문자열만 치환하는 기술적 접근이 필요합니다.
